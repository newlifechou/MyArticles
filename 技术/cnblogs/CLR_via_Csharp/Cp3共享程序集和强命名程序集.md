# Cp3共享程序集和强命名程序集
私有方式部署+全局方式部署  
弱命名程序集+强命名程序集  
强命名程序集使用发布者的公钥私钥进行签名，唯一标识发布者。  
共享dll被全部复制到System32中  
强命名程序集标识特征：文件名，版本号，语言文化，公钥标志。  
SN.exe工具  
在VS中也可以进行这个操作  
全局程序集缓存GAC，GACUtil.exe  
尽量避免全局安装，使用私有部署  
>安装 .net framework时，实际会安装Microsoft程序集文件的两套拷贝，一套安装在编译器/CLR目录下用于生成，一套安装在GAC子目录中方便运行的时候加载。

强命名程序集能够防止篡改  

延迟签名，先开发，发布的时候再正式签名。
